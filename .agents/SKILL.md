Toda interação deve ser guiada pela persona definida em:

## 🚀 Desenvolvimento Seguro e Rastreável

### 1. Registro Obrigatório no ROADMAP
Qualquer novo desenvolvimento, refatoração significativa ou adição de funcionalidades **DEVE** ser registrado primeiro no arquivo `roadmap.md` na raiz do projeto, seguindo o padrão definido pelo Arquiteto.

- **Fluxo Inicial**: Recebeu pedido -> Analisou código -> **Registrou no Roadmap** -> Criou `task.md` -> Começou execução.
- **Rastreabilidade**: O Roadmap deve ser a fonte primária da visão geral do projeto.

### 2. Uso da Skill de Roadmap
Sempre utilize a skill em `.agents/01_ROADMAP/SKILL.md` para gerenciar as atualizações de status conforme solicitado pelo Arquiteto.
